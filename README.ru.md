# SignFlow

[English](README.md) | [Русский](README.ru.md)

SignFlow — локальное браузерное приложение для подписи, проверки, шифрования и расшифрования файлов без загрузки содержимого документов на сервер приложения.

## Реализовано

- Создание отсоединённых подписей CAdES-BES (`.sig`) через КриптоПро CSP.
- Поиск сертификатов в хранилище Windows и на аппаратных токенах.
- Одна или две независимые подписи КриптоПро над одним исходным файлом.
- Проверка отсоединённых CMS/CAdES-подписей с информацией о подписанте и сертификате.
- Создание CAdES-T через явно заданную службу TSA.
- Опциональная проверка цепочки сертификатов и отзыва через настройки CRL/OCSP КриптоПро и Windows.
- Подпись RSA-сертификатом из PFX/P12 с хранением пароля только в памяти.
- Шифрование по сертификату в CMS EnvelopedData (`.p7m`) и расшифрование через КриптоПро.
- Шифрование по паролю в контейнер SignFlow AES-256-GCM (`.sfenc`) и расшифрование.
- Пакетная очередь до 100 файлов с интерфейсным ограничением 2 ГБ на файл.
- Статическая публикация в GitHub Pages и проверки CI.

## Текущие ограничения

- Браузерный RSA-адаптер не разбирает ГОСТ-контейнеры PFX/P12. Сначала импортируйте их в КриптоПро.
- Подпись через PFX/P12 сейчас поддерживает одного RSA-подписанта и SHA-256.
- Для CAdES-T нужна доступная служба TSA и может потребоваться лицензия КриптоПро TSP Client. Онлайн-проверка отзыва зависит от адресов в сертификате, системных настроек и, возможно, лицензии КриптоПро OCSP Client.
- Проверяются только отсоединённые подписи; вложенные CMS/CAdES-контейнеры пока не поддерживаются.
- Большие файлы целиком обрабатываются в памяти браузера. Потоковая обработка, Web Workers и отмена операций запланированы.
- Контейнер `.sfenc` является форматом SignFlow, а не универсальным стандартом CMS.

## Локальная разработка

Требуется Node.js 22.

```bash
npm ci
npm run dev
```

Откройте `http://localhost:3000`.

Запуск полного набора проверок:

```bash
npm run check
```

Отдельные команды: `npm run typecheck`, `npm run lint`, `npm run test` и `npm run build`.

Для проверки безопасности зависимостей используйте `npm run audit` для полного дерева и `npm run audit:prod` для production-зависимостей. CI блокирует уязвимости уровня moderate, high и critical. Dependabot еженедельно проверяет npm и GitHub Actions.

## GitHub Pages

Приложение использует статический экспорт Next.js. Workflow [`.github/workflows/deploy-pages.yml`](.github/workflows/deploy-pages.yml) собирает и публикует каталог `out/` после каждого push в `main`; также доступен ручной запуск.

В настройках репозитория выберите **Settings → Pages → Build and deployment → Source: GitHub Actions**. Адрес проекта: `https://vanitoo.github.io/Signflow/`.

## Конфиденциальность и безопасность

Документы, PFX/P12-файлы, пароли и операции с закрытыми ключами остаются на устройстве пользователя. Закрытые ключи КриптоПро используются через установленный криптопровайдер и не экспортируются. В SignFlow нет регистрации, аналитики и телеметрии.

Дополнительная информация: [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md), [docs/SECURITY_MODEL.md](docs/SECURITY_MODEL.md) и [SECURITY.md](SECURITY.md).
